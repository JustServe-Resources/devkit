
# User Journey: Add an Existing Charitable Organization to JustServe

## Target Audience
- **Primary persona:** 45-year-old woman who helps coordinate volunteers for an existing charitable organization (not the founder). Wants to find more volunteers and decides to experiment using JustServe.

## User’s Context & End Goal
- **Context:** She already supports a real-world charitable organization and she already has a JustServe account for personal use.
- **End Goal:** The organization is registered on JustServe, her account has organization admin access, and an organization page exists.

## Journey Stages
1. Start organization onboarding
2. Gain admin access
3. Create the organization page

## Actions (Action → Result)
1. User submits the **Contact Your Local Specialist** form and states she wants to add her organization → System routes the request to a local specialist for onboarding.
2. User confirms her JustServe account → Specialist can assign organization admin rights to her account.
3. User opens **Administration** from the profile menu and selects **Manage Organizations** → System displays organization management tools (admin-only).
4. User clicks **+ Add Organization**, enters the organization name, and submits → System creates the organization record.
5. User reviews and confirms **“Your Spot on JustServe”** URL → System finalizes the organization’s public URL.

## Inner Monologue
- “I’m helping with this organization, not running it—am I allowed to set this up?”
- "Why can't I just set up my organization myself? Why do I have to wait for whoever this specialist is?"
- "I hope I don't have to wait too long to hear back from the specialist."
- “Why don’t I see the Administration menu yet? Did I miss a step?”
- "There are a lot more buttons now that I'm an admin. Which one should I hit? I hope I don't mess something up."
- “This URL is permanent. I really want to make sure we choose the right one.”
- "Am I the only admin that can take care of the organization? I don't want to be stuck with all that work."

## End State (Success Indicators)

### Goal Achievement
- Organization is successfully onboarded and visible on JustServe. Results reflected in system.
- User has admin access to manage the organization page.
- Organization URL is finalized and correctly linked to the org page.

### Next Steps Clarity
- User knows she can now post and link volunteer projects to the organization.
- User understands additional admins can be added if needed.

## Alternative Paths / Edge Cases
- If the user cannot access **Administration**, onboarding continues through the local specialist.
- If a JustServe representative creates the organization page during onboarding, the user receives admin rights afterward.
