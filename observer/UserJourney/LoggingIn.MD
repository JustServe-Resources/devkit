# User Journey: Logging In (Young Person Perspective)

## Target Audience
- **Primary persona:** Young end user (teen or young adult, first-time or infrequent user)
- **Secondary variants:** Returning young user on a new device

## User’s Context & End Goal
- **Context:** The user opens an app or website on their phone or laptop and needs access to continue what they were trying to do (sign up, help, learn, or participate).
- **End Goal:** The user logs in successfully without feeling confused, stuck, or worried they did something wrong.

## Journey Stages
1. Arrive at Login Screen
2. Enter Credentials
3. Confirm Access

## Actions (Action → Result)
1. User taps **“Log in”** on the home screen → System displays the login form with email/username and password fields.
2. User enters username and password and selects **“Continue”** → System validates credentials and requests verification if required. [NEEDS VERIFICATION]
3. User completes verification (if prompted) → System signs the user in and loads the main dashboard.

## Inner Monologue
- “Okay, I think this is the right account… I hope I remember the password.”
- “Why is it asking me to verify again? Did I mess something up?”
- “Oh good, it worked. Now I can finally do what I came here to do.”

## End State (Success Indicators)
- **Goal Achievement:** User is logged in and sees the main screen they expected.
- **System Integrity:** Account loads correctly with no error messages or missing data.
- **Next Steps Clarity:** User understands they are logged in and can continue their original task (no extra login prompts).

## Alternative Paths / Edge Cases
- If the user forgets their password, they select **“Forgot password”** and are routed to account recovery. [NEEDS VERIFICATION]
- If login fails, the system displays a clear, non-blaming error message with next steps.
